# CLAUDE.md — проект «Минимум»

Персональный PWA-трекер ежедневного минимума для одного пользователя. Система различает **минимум** (неснижаемая планка, выполняется даже в худший день) и **норму** (обычный день); приложение видит только минимум. Задача — поддерживать неразрывность практики без давления: фиксировать факт и раз в неделю показывать картину. Не мотивировать, не стыдить, не развлекать.

Этот файл — конституция проекта. Он важнее любого отдельного промпта. Если задача противоречит ему — остановись и спроси архитектора, не выполняй.

## Незыблемые принципы (анти-требования)

Нарушение любого пункта = задача провалена, что бы ни просили:

- Никаких пуш-уведомлений и напоминаний.
- Никаких серий, истории и статистики прошлых дней на главном экране «Сегодня». Следы прошлого там — только нейтральная точка-маркер «вчера — пропуск» и счётчик текущей открытой недели у недельных пунктов: это операционная информация, предусмотренная спецификацией, а не статистика.
- Никаких красных состояний и стыдящих формулировок («провал», «пропущено», «серия потеряна») где бы то ни было.
- Никакой геймификации: очков, уровней, наград, конфетти, звуков, эмодзи, огня и кубков.
- Никаких графиков и дашбордов. История планки — типографическая строка, не диаграмма.
- Никаких социальных функций.
- Никакого онбординга-туториала.
- Никакого автоматического повышения минимума: любое повышение только предлагается и требует явного подтверждения.
- Пропуск — нейтральное событие (пустая ячейка), не провал.
- Самооценка раз в неделю на разборе, не ежедневно.

## Технические ограничения

- Ванильный стек без сборки: `index.html` + `styles.css` + `app.js` + `sw.js` + `manifest.json` + иконки. Никаких фреймворков, бандлеров и рантайм-зависимостей.
- Хостинг — GitHub Pages: только относительные пути (`./`), приложение должно работать из подкаталога.
- Основная платформа — iOS Safari standalone (iPhone 13 Pro Max): safe-area insets, `viewport-fit=cover`, высота через `100dvh`, тач-таргеты ≥44px, `font-size` полей ввода ≥16px (иначе iOS зумит при фокусе).
- PWA: cache-first через service worker, полный офлайн после первой загрузки.
- Язык интерфейса — русский.
- Темы: светлая и тёмная по `prefers-color-scheme`. Один акцентный цвет («эвкалипт»), нейтральная палитра, типографика-first.
- Бюджет веса без иконок ≤150 КБ; актуальный вес — в отчёте последней выполненной задачи.

## Архитектура

- `app.js` — один файл, порядок разделов: тексты «Системы» (`SYSTEM_TEXTS`) → хранилище и миграции → даты и граница дня → доменная логика → повышение планки → закрытие недели → экспорт/импорт → рендер → обработчики → запуск.
- Рендер — строковые шаблоны + `innerHTML`: `renderAll()`, `renderToday()`, `renderReview()`, `renderItems()`, `renderSystem()`. Все пользовательские строки проходят через `esc()`. Горячие пути («Сегодня»: отметка пункта, недельный счётчик) обновляют существующие DOM-узлы точечно — CSS-переходы чекбокса и планки дня должны реально проигрываться. Полная перерисовка экрана — для структурных изменений: смена вкладки, смена логического дня, открытие/закрытие форм, добавление/удаление/перестановка пунктов.
- События — делегирование на `document` через `data-act` (click / change / input).
- Хранилище — `localStorage`, неймспейс `minimum:data`, объект `store`: `items[]`, `days{}`, `weekLog[]`, `reviews[]`, `pendingRaises[]`, `draftOneChange`, `weekStart`, `settings`.
- Тестовый хук — `module.exports` в конце `app.js`. Не удалять; при добавлении доменных функций пополнять.

## Доменные инварианты (покрываются тестами)

1. **Логический день.** Время до `settings.dayBoundary` (по умолчанию 04:00) относится к предыдущему дню: отметка в 00:30 попадает во вчера. В полночь ничего не «сгорает».
2. **Скользящие недели.** Разбор доступен, когда `diffDays(today, weekStart) ≥ 7`. Окно разбора — последние 7 логических дней, включая сегодня. Пропуск разбора ничего не ломает.
3. **Закрытие недели.** `closeWeek()` пишет срез в `reviews` (perItem, trainings, oneChange, raises), очищает `weekLog`, `pendingRaises`, `draftOneChange`, ставит `weekStart = today`. Счёт тренировок в срезе — за весь открытый период недели (от `weekStart` до закрытия), сетка — последние 7 дней; это осознанное различие, период счёта фиксируется полем `weekStart` в записи review. `closeWeek()` защищён guard'ом `reviewDue()` и возвращает boolean. Последнее записанное «одно изменение» (`oneChange` последнего элемента `reviews`, непустое после trim) показывается тихой muted-строкой «Изменение этой недели: „…“» на экране разбора — и в состоянии ожидания, и в открытом разборе.
4. **Повышение планки.** Предлагается, когда после якоря `item.raiseAfter` закрыто ≥3 недель и в каждой из 3 последних закрытых недель пункт отмечен ≥6 из 7. «Принять» и «Не сейчас» ставят `raiseAfter = reviews.length + 1` — отсчёт трёх недель начинается заново.
5. **История планки.** `item.history` пишется при принятии повышения и при ручной правке значения; повторное изменение в тот же логический день заменяет последнюю запись, а не добавляет новую. Осознанная очистка значения историю не дополняет; изменение, возвращающее значение последней записи, схлопывает её — подряд одинаковых значений в истории не бывает.
6. **Экспорт/импорт.** Экспорт → очистка localStorage → импорт восстанавливает состояние полностью. Импортируемые данные всегда проходят через `migrate()`.
7. **«Не пропускай дважды».** Точка-маркер у пункта появляется, если пункт существовал вчера (`addedAt ≤ вчера`) и не был отмечен. По тапу точка раскрывает подпись «вчера — пропуск» и тихое действие «отметить»: единственная допустимая правка прошлого — установка отметки за вчера (только вчера, только через точку, только установка — не снятие). Глубже вчерашнего дня данные неизменяемы из интерфейса.
8. **Актуальность экрана.** Приложение хранит отрисованный логический день; при `visibilitychange`, `focus` и по таймеру, взведённому на ближайшую границу дня, сверяет его с `todayKey()` и при расхождении перерисовывает текущую вкладку. Любое дискретное действие (`onClick`, `onChange`) при расхождении не применяется — экран сначала перерисовывается (защита от записи не в тот день со stale-экрана). Непрерывный ввод (`onInput` черновика «одного изменения») guard'ом не прерывается: черновик принадлежит открытой неделе, а не дню.
9. **Зеркало данных.** Каждое успешное сохранение `store` дублируется в IndexedDB (БД `minimum`, один object store, один снапшот: JSON-строка + `savedAt` + `schemaVersion`) с дебаунсом и принудительным сбросом при уходе приложения в фон. При старте: валидный localStorage — источник истины, зеркало обновляется; пустой или битый localStorage при наличии снапшота — тихое восстановление из зеркала (битая строка по-прежнему сохраняется в corrupt-ключ до восстановления). Зеркало никогда не пишется до завершения этой стартовой проверки. Недоступность IndexedDB не меняет поведение приложения.

## Правила изменений

- Изменил структуру `store` → подними `SCHEMA_VERSION` и добавь шаг в `migrate()`. Данные пользователя не теряются никогда, миграции идемпотентны.
- Изменил любой файл деплоя → подними `VERSION` в `sw.js`, иначе cache-first отдаст старую версию.
- Добавил доменную функцию → экспортируй в тестовый хук и напиши тесты.
- Перед завершением задачи: `npm test` зелёный, анти-требования не нарушены, вес в бюджете.
- Не делай ничего сверх поставленной задачи. Идеи и найденные проблемы — в отчёт, не в код.
- Принятые ограничения: `dateKeyShift` неточен в час перевода стрелок (DST) — осознанно, для целевого региона неактуально. Зеркало IndexedDB — страховка от исчезновения localStorage, а не запасное хранилище при отказе записи: localStorage — единственный источник истины, при его недоступности отметки текущей сессии не сохраняются (пользователь видит баннер). Обновление приложения тихое: новая версия применяется при следующем запуске, открытая страница не перезагружается. Manifest несёт тёмные цвета бренда; цвета для каждой темы iOS берёт из двух meta theme-color с media в index.html. Таб-бар — навигация (nav + aria-current), не ARIA-tabs. Гарантированный анонс статусов скринридером — только у планки дня (постоянный узел + точечное обновление); остальные status-узлы рождаются вместе с текстом, их анонс нестабилен — осознанно. Иконки — закоммиченные артефакты: при смене палитры или мотива перегенерировать вручную (`node tools/make-icons.mjs`).

## Тесты

- Каталог `tests/`, раннер `node --test`.
- `tests/domain.test.js` — юнит-тесты доменных инвариантов через тестовый хук.
- `tests/dom.test.js` — дымовые тесты в jsdom: рендер всех четырёх экранов без исключений, базовые взаимодействия. `app.js` в jsdom загружать через `vm` в контексте window — ветка `module.exports` перехватывает обычный `require`.
- `devDependencies` допустимы только для тестов (jsdom, fake-indexeddb). Рантайм приложения — без зависимостей.
- Запуск: `npm test`.
