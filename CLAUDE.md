# CLAUDE.md — проект «Минимум»

Персональный PWA-трекер ежедневного минимума для одного пользователя. Система различает **минимум** (неснижаемая планка, выполняется даже в худший день) и **норму** (обычный день); приложение видит только минимум. Задача — поддерживать неразрывность практики без давления: фиксировать факт и раз в неделю показывать картину. Не мотивировать, не стыдить, не развлекать.

Этот файл — конституция проекта. Он важнее любого отдельного промпта. Если задача противоречит ему — остановись и спроси архитектора, не выполняй.

## Незыблемые принципы (анти-требования)

Нарушение любого пункта = задача провалена, что бы ни просили:

- Никаких пуш-уведомлений и напоминаний.
- Никаких серий, истории и статистики прошлых дней на главном экране «Сегодня». Следы прошлого там — только нейтральная точка-маркер «вчера — пропуск» и счётчик текущей открытой недели у недельных пунктов: это операционная информация, предусмотренная спецификацией, а не статистика. Единственное точечное исключение из «никаких серий» — серия недель привычки (решение владельца от 19.07.2026, см. «Принятые ограничения»).
- Никаких красных состояний и стыдящих формулировок («провал», «пропущено», «серия потеряна») где бы то ни было.
- Никакой геймификации: очков, уровней, наград, конфетти, звуков, эмодзи, огня и кубков.
- Никаких графиков и дашбордов. История планки — типографическая строка, не диаграмма.
- Никаких социальных функций.
- Никакого онбординга-туториала.
- Никакого автоматического повышения минимума: любое повышение только предлагается и требует явного подтверждения.
- Пропуск — нейтральное событие (пустая ячейка), не провал.
- Самооценка раз в неделю на разборе, не ежедневно.

## Технические ограничения

- Ванильный стек без сборки: `index.html` + `styles.css` + `app.js` + `sw.js` + `manifest.json` + иконки. Никаких фреймворков, бандлеров и рантайм-зависимостей.
- Хостинг — GitHub Pages: только относительные пути (`./`), приложение должно работать из подкаталога.
- Основная платформа — iOS Safari standalone (iPhone 13 Pro Max): safe-area insets, `viewport-fit=cover`, высота через `100dvh`, тач-таргеты ≥44px, `font-size` полей ввода ≥16px (иначе iOS зумит при фокусе).
- PWA: cache-first через service worker, полный офлайн после первой загрузки.
- Язык интерфейса — русский.
- Темы: светлая и тёмная по `prefers-color-scheme`. Один акцентный цвет («эвкалипт»), нейтральная палитра, типографика-first.
- Бюджет веса без иконок ≤150 КБ; актуальный вес — в отчёте последней выполненной задачи.

## Деплой

Публичный репозиторий GitHub `xbaox/minimum`. GitHub Pages — из ветки `main`, каталог `/` (root), файл `.nojekyll` в корне. Релиз = push в `main` с поднятым `VERSION` в `sw.js`; CI (`npm test`) обязан быть зелёным. Пользовательских данных в репозитории нет — они живут только в localStorage и IndexedDB устройства.

## Архитектура

- `app.js` — один файл, порядок разделов: тексты «Системы» (`SYSTEM_TEXTS`) → хранилище и миграции → даты и граница дня → доменная логика → повышение планки → закрытие недели → экспорт/импорт → рендер → обработчики → запуск.
- Рендер — строковые шаблоны + `innerHTML`: `renderAll()`, `renderToday()`, `renderReview()`, `renderItems()`, `renderSystem()`. Все пользовательские строки проходят через `esc()`. Горячие пути («Сегодня»: отметка пункта, недельный счётчик) обновляют существующие DOM-узлы точечно — CSS-переходы чекбокса и планки дня должны реально проигрываться. Полная перерисовка экрана — для структурных изменений: смена вкладки, смена логического дня, открытие/закрытие форм, добавление/удаление/перестановка пунктов.
- События — делегирование на `document` через `data-act` (click / change / input).
- Пять вкладок: Сегодня · Привычки · Система · Разбор · Пункты. Экран «Привычки» показывает сегодняшнее действие плюс полосу текущей недели и серию как тихую справку (решение владельца от 19.07.2026); остальные анти-требования действуют на нём полностью; внизу — кредо-строка «Не спеши — доверься накопительному эффекту.»
- Хранилище — `localStorage` (схема v6), неймспейс `minimum:data`, объект `store`: `items[]`, `days{}`, `weekLog[]`, `reviews[]`, `pendingRaises[]`, `draftOneChange`, `weekStart`, `settings`.
- Тестовый хук — `module.exports` в конце `app.js`. Не удалять; при добавлении доменных функций пополнять.

## Доменные инварианты (покрываются тестами)

1. **Логический день.** Время до `settings.dayBoundary` (по умолчанию 04:00) относится к предыдущему дню: отметка в 00:30 попадает во вчера. В полночь ничего не «сгорает».
2. **Календарные недели.** Неделя — понедельник–воскресенье в логических днях (принадлежность дня неделе определяется его логическим ключом). `weekStartOf(dayKey)` — понедельник недели этого дня; `currentWeekStart()` — `weekStartOf(todayKey())`; недели существуют только с `settings.calendarSince`. Разбор предлагается только за последнюю завершённую неделю: `reviewDue()` истинно, когда `previousWeekStart = currentWeekStart − 7 дней` существует, ≥ `calendarSince` и не встречается среди `week` закрытых разборов. Окно разбора — ровно эта завершённая неделя; сегодняшний день в окно не входит. Пропуск разбора ничего не ломает: непросмотренные недели тихо проходят, их отметки остаются в `days{}` и в экспорте.
3. **Закрытие недели.** `closeWeek()` защищён guard'ом `reviewDue()` и возвращает boolean. Срез: `{week: понедельник разобранной недели, perItem за её 7 дней, trainings — записи weekLog той же недели, oneChange, raises, params}`; после записи чистятся `pendingRaises`, `draftOneChange` и `paramDecided`, из `weekLog` удаляются записи старше `currentWeekStart()`. Счётчик «Тренировка N из…» на «Сегодня» считает записи `weekLog` текущей календарной недели и обнуляется сменой недели, а не закрытием разбора; «отменить последний» действует только на записи текущей недели. Поле `weekStart` в старых записях `reviews` — историческое (скользящая эпоха), новые записи несут `week`. Последнее записанное «одно изменение» (`oneChange` последнего элемента `reviews`, непустое после trim) показывается тихой muted-строкой «Изменение этой недели: „…“» на экране разбора — и в ожидании, и в открытом разборе.
4. **Повышение планки.** Предлагается, когда после якоря `item.raiseAfter` закрыто ≥3 недель и в каждой из 3 последних закрытых недель пункт отмечен ≥6 из 7. «Принять» и «Не сейчас» ставят `raiseAfter = reviews.length + 1` — отсчёт трёх недель начинается заново.
5. **История планки.** `item.history` пишется при принятии повышения и при ручной правке значения; повторное изменение в тот же логический день заменяет последнюю запись, а не добавляет новую. Осознанная очистка значения историю не дополняет; изменение, возвращающее значение последней записи, схлопывает её — подряд одинаковых значений в истории не бывает.
6. **Экспорт/импорт.** Экспорт → очистка localStorage → импорт восстанавливает состояние полностью. Импортируемые данные всегда проходят через `migrate()`.
7. **«Не пропускай дважды».** Точка-маркер у пункта появляется, если пункт существовал вчера (`addedAt ≤ вчера`) и не был отмечен. По тапу точка раскрывает подпись «вчера — пропуск» и тихое действие «отметить»: единственная допустимая правка прошлого — установка отметки за вчера (только вчера, только через точку, только установка — не снятие). Глубже вчерашнего дня данные неизменяемы из интерфейса.
8. **Актуальность экрана.** Приложение хранит отрисованный логический день; при `visibilitychange`, `focus` и по таймеру, взведённому на ближайшую границу дня, сверяет его с `todayKey()` и при расхождении перерисовывает текущую вкладку. Любое дискретное действие (`onClick`, `onChange`) при расхождении не применяется — экран сначала перерисовывается (защита от записи не в тот день со stale-экрана). Непрерывный ввод (`onInput` черновика «одного изменения») guard'ом не прерывается: черновик принадлежит открытой неделе, а не дню.
9. **Зеркало данных.** Каждое успешное сохранение `store` дублируется в IndexedDB (БД `minimum`, один object store, один снапшот: JSON-строка + `savedAt` + `schemaVersion`) с дебаунсом и принудительным сбросом при уходе приложения в фон. При старте: валидный localStorage — источник истины, зеркало обновляется; пустой или битый localStorage при наличии снапшота — тихое восстановление из зеркала (битая строка по-прежнему сохраняется в corrupt-ключ до восстановления). Зеркало никогда не пишется до завершения этой стартовой проверки. Недоступность IndexedDB не меняет поведение приложения.
10. **Две области.** Каждый пункт несёт `area: 'min' | 'habit'`. Минимум — неснижаемая планка: планка дня, «День закрыт», недельный счётчик тренировок и механика повышения принадлежат только area `min`. Привычки — программа роста: ежедневные привычки (`type daily`, area `habit`, без значения в формах) и недельные параметры (`type param`, только area `habit`: `pkind 'time'|'number'`, `pvalue` — минуты суток для time, число для number, `pstep` — целое со знаком, `unit` для number; `pkind` фиксируется при создании и не редактируется). У привычек своя планка дня на вкладке «Привычки». Параметры не участвуют ни в одной планке и не имеют ежедневных отметок; на вкладке — пассивная строка с текущим порогом. Порог меняется шагом только в открытом разборе: одно решение на параметр за разбор — «шаг» применяет `pstep` немедленно и пишет history, «оставить» фиксирует отказ; решения хранятся в `store.paramDecided` с привязкой к `week` разбираемой недели; решения чужой недели игнорируются, `closeWeek` переносит в срез `review.params [{id, from, to|null}]` только решения разобранной недели и очищает `paramDecided` целиком. Точка-маркер и ретро-отметка «вчера» работают для ежедневных пунктов обеих областей. Готовность к новой привычке — информационная muted-строка в разборе, когда в 2 последних закрытых неделях каждая активная привычка выполнила норму (≥ `normPerWeek`, инвариант 11); никаких автоматических действий и кнопок.
11. **Норма и серия привычки.** Каждая привычка (`type daily`, area `habit`) несёт `normPerWeek` — целое ∈ [1..7], по умолчанию 7; только у привычек, невалидное значение приводится к ближайшему допустимому. Неделя выполнена, если отметок привычки в её календарной неделе ≥ `normPerWeek`. Серия — число подряд выполненных завершённых недель, считая назад от последней завершённой; текущая неделя не входит (сегодняшний тап серию не меняет); первый провал обрывает счёт; недели существуют только с `calendarSince`; недели до создания привычки и неполная неделя создания спецобработки не имеют — не набрав норму, они просто обрывают счёт. Серия и полоса недели — чистые функции от `days{}`, `normPerWeek` и календаря: разбор и `closeWeek` на них не влияют; смена нормы ретроактивно пересчитывает серию по всей истории. Дневные отметки привычек изменяемы через UI только за сегодня (круг) и за вчера (ретро-механизм инварианта 7); глубже и в будущее — никак.

## Правила изменений

- Изменил структуру `store` → подними `SCHEMA_VERSION` и добавь шаг в `migrate()`. Данные пользователя не теряются никогда, миграции идемпотентны.
- Изменил любой файл деплоя → подними `VERSION` в `sw.js`, иначе cache-first отдаст старую версию.
- Добавил доменную функцию → экспортируй в тестовый хук и напиши тесты.
- Перед завершением задачи: `npm test` зелёный, анти-требования не нарушены, вес в бюджете.
- Не делай ничего сверх поставленной задачи. Идеи и найденные проблемы — в отчёт, не в код.
- Мутационные проверки и любые деструктивные git-операции (`checkout --`, `reset --hard`, переписывание истории) — только в изолированной копии рабочей директории, никогда в живом дереве.
- Раздел «Сделано» отчёта ведётся строго по номерам пунктов исходного промпта; каждый пункт получает явный вердикт: сделан / не сделан / отменён решением архитектора.
- Принятые ограничения: `dateKeyShift` неточен в час перевода стрелок (DST) — осознанно, для целевого региона неактуально. Зеркало IndexedDB — страховка от исчезновения localStorage, а не запасное хранилище при отказе записи: localStorage — единственный источник истины, при его недоступности отметки текущей сессии не сохраняются (пользователь видит баннер). Обновление приложения тихое: новая версия применяется при следующем запуске, открытая страница не перезагружается. Manifest несёт тёмные цвета бренда; цвета для каждой темы iOS берёт из двух meta theme-color с media в index.html. Таб-бар — навигация (nav + aria-current), не ARIA-tabs. Гарантированный анонс статусов скринридером — только у планки дня (постоянный узел + точечное обновление); остальные status-узлы рождаются вместе с текстом, их анонс нестабилен — осознанно. Иконки — закоммиченные артефакты: при смене палитры или мотива перегенерировать вручную (`node tools/make-icons.mjs`). Ретро-отметка, сделанная в день закрытия недели, ложится в `days{}` (и в экспорт), но в уже замороженный срез `reviews` не попадает: срезы неизменяемы, первичен факт. Переход со скользящих недель: `calendarSince` ставится миграцией в понедельник дня миграции (если это понедельник) или ближайший следующий; до этой даты разбор недоступен, счётчик тренировок доживает от прежнего `store.weekStart`. Последняя скользящая неделя и дни до `calendarSince` среза не получают — их отметки остаются в `days{}` (расширение принципа «срезы неизменяемы, первичен факт»). Решение владельца от 19.07.2026: серия недель привычки — точечное исключение из «никаких серий», тихая ретроспективная справка muted-текстом на «Привычках» и в разборе — операционная информация программы роста, по аналогии с недельным счётчиком; запрещёнными остаются празднования, эмодзи, цветовое давление, сортировка по серии, призывы «не прервать» и любые уведомления. Миграция v6 аддитивна: `days{}` и `reviews[]` не изменяются.
